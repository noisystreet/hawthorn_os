# 山楂 / hawthorn 微内核：模块与设计

> **[English](./en/KERNEL.md)** — English mirror of this document.

本文档描述 **山楂（hawthorn）微内核** 的内部模块划分、对象模型、接口边界与实时相关设计。**系统整体分层**见 [架构.md](./架构.md)；本文默认读者已了解微内核「小内核 + 用户态服务」的基本思想。

## 2. 设计目标与边界

### 2.1 目标

- **最小可信计算基（TCB）**：能直接导致任意内存读写或绕过隔离的代码，尽量收敛在 **小体积、可审计** 的内核中。
- **故障隔离**：驱动或协议栈崩溃可通过 **进程/地址空间边界** 限制影响，可重启单服务而不必整机复位（策略由监督服务实现）。
- **统一 IPC**：用户态与内核、服务与服务之间使用 **同一套消息原语**（或少量正交扩展），便于形式化与测试。
- **可预测实时**：为硬实时线程保留 **短路径**：调度决策、IPC 快速路径、中断到线程的递交延迟可分析。

### 2.2 内核职责边界（做什么 / 不做什么）

**内核内应包含：**

- 线程抽象与 **抢占式调度**（及与优先级相关的基本策略）。
- **地址空间** 与 **映射** 的建立/切换（MPU/MMU 视平台而定）。
- **能力（Capability）** 或等价 **引用监视** 机制：内核对象句柄的创建、传递、撤销与权限检查。
- **IPC**：端点、同步消息传递、（可选）异步通知/信号量等最小集合。
- **中断/异常** 的硬件入口与 **向线程或服务递交**（内核不实现具体设备协议）。
- **时钟节拍 / 定时器** 的基础设施（墙上时间与单调时钟策略可与用户态时间服务协同）。
- **系统调用入口** 与 **trap 上下文** 切换。

**内核外（用户态服务）应包含：**

- 具体 **设备驱动**（GPIO、UART、DMA、CAN、以太网 MAC 等）。
- **总线类协议栈**（TCP/IP、USB 栈等）与 **文件系统**。
- **设备枚举、电源策略、OTA 业务逻辑** 等（可由一个或多个服务拆分）。

**灰色地带（实现阶段需写清）：**

- **内核映射的 MMIO 窗口** 是否仅由内核建立后通过能力交给驱动服务，还是允许服务自建映射（由内存管理器服务代理）。须在移植指南中固定一种模型并文档化。

### 2.3 首发平台硬件约束（香橙派 5 / RK3588）

首发板卡为 **香橙派 5**，SoC **RK3588**，内核模块设计需与下列硬件事实对齐（细节以 **RK3588 TRM** 与板级原理图为准）：

| 领域 | 对内核模块的直接影响 |
|------|----------------------|
| **ISA** | **AArch64**：`trap`/`syscall` 使用 **SVC/HVC/SMC** 与 ARMv8-A 异常模型；上下文保存含 **SIMD/FP** 与否由 ABI 与 `EL0` 使用约定决定。 |
| **MMU** | `mem` 以 **页表**（Granule、TG0/TG1 等）管理 **`EL0`/`EL1` 映射**；无 Cortex-M 式 MPU 主路径。 |
| **中断** | `irq` 对接 **GIC**（SPI/PPI/SGI）；多核时配置 **GIC Redistributor** 每核、**IPI** 经 SGI 或等价机制。 |
| **计时** | `time` 优先绑定 **ARM Generic Timer**（`CNTPCT` / `CNTVCT`、tick 与 one-shot 策略与 `sched` 协同）。 |
| **多核** | `sched` 预留 **SMP**：RK3588 为 **4×A76 + 4×A55**，需明确 **集群/CPU 拓扑**、负载迁移与 **big.LITTLE** 策略（与产品实时目标一起收敛）。 |
| **DMA / 一致性** | 设备 DMA 与 **D-cache 维护** 在契约上落在 **HAL + 内存/驱动服务**；内核 `mem` 提供映射与权限时，须避免设备与 CPU 看到 **不一致缓存视图**（具体刷法与内存属性见 BSP/HAL 文档）。 |

Bootloader / **TF-A** 移交的 **异常级（EL）**、入口 PC、设备树指针等，由 `boot` 与 `bsp/orangepi5-rk3588/` 固定为 **版本化启动信息块**，与 [架构.md §2.5](./架构.md) 一致。

## 3. 内核模块分解

以下模块在逻辑上正交，实现上可映射为 `kernel/` 下多个 crate 或子模块（如 `sched`、`ipc`、`mem`、`syscall`）。

### 3.1 引导与早期初始化（`boot`）

- **入口契约**：Bootloader 传入 **启动信息块**（内存布局、保留区、设备树/板级描述指针、启动镜像槽位等），版本化结构体并校验魔数。
- **阶段划分**：关中断下的 **极短序列**（栈、BSS、CPU 特性、**MMU** 最小使能；若平台另有 **MPU** 则一并配置）→ 建立首个内核线程与 **根能力空间** → 跳转到可调度上下文。RK3588 主路径为 **EL1 页表**，见上文 **§2.3**。
- **首个用户任务**：创建 **根任务（init）** 或等价引导服务，由其加载/启动其他服务（名称服务、驱动等）。

**QEMU `virt` 最小着陆（M2，当前实现）**：用于 CI 与本地冒烟的 AArch64 裸机路径在 **`hawthorn_kernel`** 内收敛。入口 **`_start`**（段 `.text.boot`），若 QEMU 在 EL2 进入则降级到 EL1 并禁用 MMU、刷新 TLB；初始 **`SP = __stack_top`**，由 [`kernel/link-qemu_virt.ld`](../kernel/link-qemu_virt.ld) 定义：`RAM` 起 **`0x4000_0000`**、长 **128 MiB**，**`__stack_top = ORIGIN(RAM) + LENGTH(RAM) - 16`**（向下生长的满降栈）。汇编 **`bl kernel_main`**；**`kernel_main`** 在 `hawthorn_kernel::boot_qemu_virt`（关中断假设下的极短序列：BSS 清零 → **PL011** @ **`0x9000_0000`** 初始化 → **异常向量表** `trap::init()` → **GICv3** `gic::init()` → **IRQ 分发** `irq::init()` → 打印 banner）。可执行目标 **`hawthorn_kernel_qemu_virt`** 需 **`cargo build -p hawthorn_kernel --features bare-metal --target aarch64-unknown-none`**；**`hawthorn_qemu_minimal`** 与此共享同一链接脚本路径（见 [移植.md](./移植.md) §1.2）。**Panic** 经同一 UART 输出 **`hawthorn_kernel: panic\n`**（`#[panic_handler]` 在 bin crate 根）。异常级：`_start` 显式处理 EL2→EL1 降级，确保在 EL1 运行并禁用 MMU。

### 3.2 内核对象模型（`objects`）

建议支持的最小对象类型（名称可按实现调整）：

| 对象 | 作用 |
|------|------|
| `Thread` | 执行上下文、优先级、栈、寄存器保存区。 |
| `Task` / `Process` | 可选分组：若干线程共享一地址空间与能力表。 |
| `AddressSpace` | 与硬件 ASID/域绑定；持有映射表或指向页表根。 |
| `Endpoint`（或 `Port`） | IPC 的一端；权限与配额挂在能力上。 |
| `Notification` / `IrqControl` | 中断完成与线程唤醒的桥梁（二选一或组合）。 |
| `CNode` / 能力槽位表 | 能力的存储与派生关系。 |

**派生与权限**：能力支持 **缩小权限的派生**（如只读映射、只收发不能转发），撤销时级联失效或引用计数策略需在实现说明中固定。

### 3.3 调度子系统（`sched`）

- **策略**：单核阶段采用 **固定优先级抢占式调度（FP）**；同优先级 **FIFO** 或 **轮转** 可配置。
- **就绪队列**：按优先级分桶；避免在 ISR 内做复杂调度，仅 **置位/入队** 由内核线程或尾程处理。
- **迁移（SMP 预留）**：线程绑定 CPU 或 **全局队列** 策略二选一时，文档化 **最坏迁移开销** 与锁持有时间。
- **与 IPC 关系**：阻塞式接收导致 **阻塞态**；发送非阻塞或带超时由 IPC 层与调度协同。

**当前实现（M4）**：`task.rs` 为 **固定优先级抢占式调度**：ARM Generic Timer tick（`timer.rs`）调用 `task::tick()` 递减当前任务时间片，耗尽时置 `NEED_RESCHEDULE`，在 `trap` 的 IRQ 返回路径上调用 `schedule()`；**同优先级**在时间片耗尽时按任务下标 **轮转**。`yield_now()` 仍可用于主动让出。阻塞原语包括 **`sleep(ms)`**（按 tick 唤醒）、**`block` / `unblock`**（供 IPC 等使用）。TCB 含 5 种状态（Unused / Ready / Running / Blocked / Exited），`TaskContext` 保存 callee-saved 寄存器（x19–x30）及 **DAIF**，`context_switch` 汇编用 x2 中转 SP（AArch64 不允许 `str sp`）。内核任务经 `task_trampoline` 启动；EL0 用户任务经 `user_task_trampoline`；返回后走 `task_exit`。调度策略的纯函数切片见 `task_policy.rs`（`#[cfg(test)]`）。

### 3.4 IPC 子系统（`ipc`）

- **同步消息**：`call` / `reply` 或 **rendezvous** 模型；消息负载 **有上界**（小消息寄存器传递 + 可选页内共享窗口由能力授权）。
- **能力传递**：随消息 **转移或复制** 能力槽位，避免伪造句柄。
- **快速路径**：热路径避免不必要拷贝与分配；关键数据结构 **cache 友好** 并对齐分析。
- **背压**：队列深度、未处理 `reply` 数量等 **配额** 防 DoS；与 [架构.md](./架构.md) 中资源上限一致。

### 3.5 内存子系统（`mem`）

- **内核自身**：静态或 **启动期一次性** 分配；运行期热路径避免通用堆。
- **用户地址空间**：映射建立、unmap、**DMA 一致性** 钩子（与 HAL 约定：刷 cache / 非 cache 区）在 **内存管理器服务** 中实现时，内核仅提供 **映射原语** 与 **能力校验**。
- **物理内存账册**：帧分配可在内核简化实现，或委托 **pager/内存服务**；无论哪种，**双重映射与 W^X** 策略须明确。

### 3.6 中断与异常（`irq` / `trap`）

- **硬件向量**：仅负责保存上下文、 **_ack 与分类**、将事件关联到 **Notification** 或 **IPC 目标**。
- **RK3588**：经 **GIC** 分发；SMP 下注意 **SPI 目标核** 与 **负载均衡** 是否由内核配置或由驱动服务持有能力后配置（实现前选定一种责任划分）。
- **下半部**：长逻辑不在 ISR；通过 **唤醒高优先级线程** 或 **向驱动服务发消息** 完成。
- **栈与嵌套**：文档化 **每级异常栈** 与最大嵌套深度，防止栈溢出。

**当前实现（M2）**：`trap.rs` 设置 `VBAR_EL1`，16 槽向量表以 `global_asm!` 内联。EL1 IRQ 分发到 `irq::dispatch()`（而非桩）。`gic.rs` 实现 GICv3 单核初始化（Distributor / Redistributor SGI base page / CPU Interface），`ack()` 读 `ICC_IAR1_EL1`，`eoi()` 写 `ICC_EOIR1_EL1`。`irq.rs` 维护 1020 槽 handler 表，`register()` 同时在 GIC 侧使能中断。第一个注册的 IRQ handler 为 Timer PPI 30。

### 3.7 时间子系统（`time`）

- **节拍源**：硬件定时器 + tickless（若启用）对 **调度时钟** 的影响需评估。
- **超时**：IPC 与阻塞原语上的 **相对/绝对超时** 与时钟漂移策略。
- **与用户态**：授时、NTP/PTP 等仍在服务中；内核提供 **读时钟** 与 **定时器对象**（若纳入最小对象集）。

**当前实现（M3）**：`timer.rs` 驱动 EL1 Physical Timer（PPI IRQ 30），从 `CNTFRQ_EL0` 读取频率（QEMU virt 上为 62.5 MHz），默认 10 ms 周期 tick（写入 `CNTP_TVAL_EL0`）。`handle_irq()` 递增单调 `TICK_COUNT`，每 100 个 tick 打印一次。IRQ 在 `kernel_main` 末尾通过 `msr daifclr, #2` 使能。当前为纯周期 tick，无 tickless；M4 将添加等待队列和 `task::sleep(ms)` 阻塞原语。

### 3.8 系统调用接口（`syscall`）

- **ABI 稳定面**：系统调用号、参数寄存器约定、错误码枚举；**用户态存根** 与内核路径共享同一头文件或生成代码（仓库为 **`hawthorn_syscall_abi`** crate）。
- **校验**：所有用户指针经 **能力+范围** 校验后再访问；避免 **TOCTOU**（可先拷贝到小内核缓冲或配合硬件保护）。
- **调试**：可选 **跟踪点** 与 **慢路径日志**（编译期开关）。

**当前实现（QEMU virt）**：`trap` 对 **EL1 与 EL0** 的 SVC（`ESR.EC == 0x15`）均调用 `syscall::dispatch`（`x8` = 调用号，`x0`–`x5` 为参数，`x0` 返回；错误为 **负 errno** 风格，见该 crate 文档）。已实现调用号包括 **`SYS_WRITE`**、**`SYS_READ`（桩，返回 `ENOSYS`）**、**`SYS_YIELD`**、**`SYS_GETPID`**、**`SYS_EXIT`**、**`SYS_SLEEP`**、端点 MVP（create/destroy/call/recv/reply）及 **`SYS_ABI_INFO`**：`hawthorn_syscall_abi::abi_info_word()` 约定为 **低 32 位 = `ABI_VERSION`**、**高 32 位 = `ABI_CAP_*` 掩码**（与 errno 编码无关）。**权威定义与号段规则**见 [系统调用ABI.md](./系统调用ABI.md)（DRAFT-1.0）。

**模拟 EL0 用户态（MVP）**：`task::create_user` 为任务分配 **独立 TTBR0 用户页表**（自内核页表克隆），将链接段 **`.user_program`** 映射到 **`user_layout::USER_CODE_BASE`（`0x1000`）**；用户栈为 **单页**，顶为 **`user_layout::USER_STACK_TOP`（`0x8000`）**，**`USER_STACK_BOTTOM..TOP`** 已映射，**其下另有一页 guard（未映射）**；**代码页与用户栈页之间的 VA 区间未映射（空洞）**。首次调度经 **`user_return`** 以 **AArch64 EL0t** 执行内嵌程序。`SYS_WRITE` 等对 **用户缓冲** 必须 **完全落在** 已映射 **代码窗** 或 **栈窗** 之一内（见 **`kernel/src/user_layout.rs`** 与 **`syscall`**），否则返回 **`EFAULT`**。`create_user` 的入口参数须与上述固定布局一致（MVP 约束）。

## 4. 与用户态服务的契约

- **init**：持有初始能力集，负责 **服务注册、重启策略、看门狗协作**。
- **驱动服务**：通过 **IPC** 向客户端暴露 **设备会话**；DMA 缓冲通过 **能力授权的映射** 或 **共享页** 传递。
- **命名/发现**：服务名到端点的解析可在独立 **名称服务** 中实现，内核可提供 **引导端点** 硬编码或配置表。

依赖方向：**用户态服务不链接内核符号**；仅通过 **syscall + 稳定 ABI** 交互（便于替换内核实现与模糊测试）。

## 5. 实时性要点（微内核语境）

- **控制回路**：关节/电流环放在 **高优先级线程**；与驱动通信使用 **短消息** 或 **预映射共享区 + 信号**，避免大块拷贝。
- **IPC 延迟**：将 **关键路径** 控制在内核可证明的指令与 cache miss 预算内；必要时为 **特定调用号** 提供快速路径。
- **隔离与抢占**：低优先级服务不得 **长时间关抢占**；内核锁 **分区** 或 **无锁结构** 降低优先级反转面。

---

## 6. 测试与验证建议

- **调度 + IPC 状态机**：在 host 上用 **模拟硬件** 做单元与属性测试。
- **模糊 syscall**：对用户态传入的随机序列做 **内核硬化** 回归。
- **最坏情况分析（WCET）**：对发布版本做一次 **测量 + 文档化**（至少关键板型）。

## 7. 与仓库目录的对应（建议）

```
kernel/
├── boot/          # 早期启动
├── objects/       # 对象与能力
├── sched/         # 调度
├── ipc/           # IPC
├── mem/           # 地址空间与映射原语
├── irq/           # 中断递交
├── time/          # 时钟与定时器
└── syscall/       # 陷入与分发
```

用户态服务建议置于 `servers/`（或等价目录），与 `kernel/` **无 Cargo 依赖反向链接**。
