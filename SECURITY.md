# 安全策略

## 支持的版本

**山楂（hawthorn）** 仍处于 **早期开发** 阶段，尚无正式版本划分。若未来发布带版本号的制品，将在此列出仍接收安全更新的版本。

## 报告漏洞

若你认为发现了 **山楂（hawthorn）** 中的安全漏洞（例如内核隔离被绕过、权限检查错误等），请 **不要** 在公开 GitHub Issue 中披露细节。

建议做法：

1. 使用托管平台提供的 **「Private vulnerability reporting」**（若已开启），或  
2. 向仓库维护者发送 **私信**（邮件等由维护者在项目主页公布时补充）。

报告中请尽量包含：复现环境（板卡、SoC、commit）、影响范围与是否已有利用思路。

## 披露节奏

维护者在确认问题后会协调 **修复与披露** 时间线；在修复可用前，请避免公开可利用细节。
